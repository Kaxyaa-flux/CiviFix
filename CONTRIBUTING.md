# Contributing to CiviFix

First off, thank you for considering contributing to CiviFix! It's people like you that make CiviFix such a great tool for building smarter and more transparent cities.

## How Can I Contribute?

### Reporting Bugs

If you find a bug, please create an issue on GitHub. Before creating a new issue, please check if a similar issue already exists.

When creating an issue, include:
* A clear and descriptive title
* Steps to reproduce the bug
* Expected and actual behavior
* Your environment (OS, Browser, Node version)

### Suggesting Enhancements

We're always looking for ways to make CiviFix better! If you have an idea, feel free to open an issue to discuss it before you start working on it.

### Pull Requests

1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes.
4. Make sure your code lints (run `npm run lint`).
5. Update the documentation if needed.
6. Issue that pull request!

## Development Setup

1. Clone your fork: `git clone https://github.com/your-username/CiviFix.git`
2. Install dependencies: `npm install`
3. Setup `.env` file (copy `.env.example`)
4. Start the development server: `npm run dev`

## Commit Messages

We use the [Conventional Commits](https://www.conventionalcommits.org/) convention for our commit messages.

Examples:
* `feat: add new map filtering`
* `fix: resolve issue with image upload`
* `docs: update setup instructions`

Thank you for helping us build CiviFix!
